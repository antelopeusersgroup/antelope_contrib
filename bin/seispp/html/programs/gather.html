gather

Usage:

gather -i key1 key2 ... -s key1 key2 ... [-text] < in > out

Algorithm:

Assembles a file of serialized ThreeComponentSeismogram objects read from stdin into one or more ThreeComponentEnsemble objects that are serialized and written to stdout.  Grouping that defines the ensemble is defined by exact matches of integer and/or string keys.   A new ensemble is created and written to output whenever any of the keys change.  It is thus assumed the file has been properly sorted before running this program.

-s should precede the list of string keys.
-i should precede the list of integer keys.
Note only integer keys, only string keys, and mixes of integer and string keys are all allowed.

Options:

-text
Switch to text format input and output.  Default is binary format serialized data.  The -binary argument must occur as the first argument, the last argument, or between the -i and -s key lists.  

Limitations and Caveats

  1. If the gather keys given are not consistent with the order of the file the results will likely be garbage. 
  2. If the gather keys never change in a large file a memory fault can be generated as the program has not safety valve for large ensembles created in error.
  3. Only exact match grouping is currently supported.   Defining real number matches, for example, would require range matching that would require a different algorithm.   That type of functionality might be easier to implement with a prefilter that does more elaborate metadata processing and assigns a new integer key that can be handled correctly with this program.
