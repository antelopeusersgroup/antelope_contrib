linearmoveout

Usage:

linearmoveout [-vr v_reduce -t0 timeshift --help -text] < infile > outfile

Algorithm:

Applies a linear moveout correction with a specified velocity and time shift.  The time shift for each seismogram is computed by extracting the real-valued attribute with the (frozen) name offset.  The computed time shift is t0+v_reduce*offset where t0 and v_reduce are optional arguments with defaults (see below).  Units are not specified but it is assumed time has units of seconds.  Hence, the distance units can be anything convenient (you can use furlongs per second if you want as long as the offset attribute has units of furlongs). 

Only TimeSeriesEnemble objects are currently supported as input and output.

Options:

-vr
Specify the reducing velocity (moveout velocity) for the linear time shift.   Default is 6000.0.

-t0
Specify the offset time shift to use for the correction.   Assumed in units of seconds.  Default is 0.0.

--help
Print the usage line.

-text
Write the output in text format.  Default is a binary serialized file.
