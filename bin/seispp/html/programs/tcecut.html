tcecut

Usage:

tcecut tmin tmax  < in > out [-v --help -text]

Algorithm:

This program can be used to apply a reduced time window to seismograms stored in ThreeComponentEnsemble objects serialized to the input file read from stdin.  It is conceptually equivalent to the Seismic Unix program suwind run in time windowing mode.  

The time interval to extract is defined by the time range tmin<=t<=tmax.   For more than a single ensemble this always requires the data be in a relative time standard.    The two current examples are multichannel shot gathers where time 0 is the shot time and the arrival time reference used, for example, in dbxcor.  It is possible to use epoch times for tmin and tmax, but I can think of no example where this would be appropriate for more than a single ensemble.  The only example of this I can conceive is an gather formed from a set of stations that span a common absolute time window.  

Options:

-text
Write the output in text format.  Default is a binary serialized file.

--help
Echo the usage line. 

-v
More verbose output to stderr.
