AGC

Usage:

agc windlength [-g gainfunction] < infile > outfile

Algorithm:

This is a three-component agc operator.   The algorithm was stolen from seismic unix, but extended to three-component data.   That means the moving rms window used to define the gain function is computed from the vector magnitude of each sample rather than the scale rms used for stock seismic reflection data handled by seismic unix. 

Options:

The required argument windlength defines the agc window length in seconds.   No sanity checks are made on this parameter.

If the  -g option is specified the computed gain functions for each seismogram are saved as serialized TimeSeriesEnsemble objects to the file defined by the argument that follows the -g flag (gainfunction in the usage line).
