subset_streamfile

Usage:

subset_streamfile key:type ( -eq val | -range minval maxval | -min minval -max maxval ) [-objt object_type --help -text] < infile > outfile

Algorithm:

This program will subset a seispp serialized stream file of all supported types using a range or equality test against an attribute the program assumes is stored in each object's Metadata (header) space.  This program can be viewed as a functional equivalent to the seismic unix program suwind.  It is, however, more general in some ways and more restricted in others.   The output file will contains a reduced set of objects that satisfy the subset condition.

The program support integer, real, and string keys.   You can specify a pure equality test (sensible, of course, only for integer or string keys) or a range test.   Use the -eq argument to specify an equality test.  Use -min to select data greater than minval.  Similarly use -max arguments to specify data less than maxval.   To specify an interval use the -range argument.  Subset for -range  is minval<key<maxval.

The type of the key is specified by a special syntax for argument 1 borrowed from GMT.  Accepted values of the type field are defined by the following table:
Data type
 Allowed values
integer
 int, INT, long
real
 real, float, double
string
 string, STRING


Options:

-objt object_type
Set the type of objects the input file is expected to contain.   Allowed options are:  ThreeComponentSeismogram (default), ThreeComponentEnsemble, TimeSeries, TimeSeriesEnsemble, and PMTimeSeries.   Note that for ensembles the test is applied to the ensemble metadata not the single seismogram metadata. 

-text
Write the output in text format.  Default is a binary serialized file.

--help
Echo the usage line.
