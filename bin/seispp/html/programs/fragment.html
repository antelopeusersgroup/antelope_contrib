fragment

Usage:

fragment basename [-i infile -dir outdir -binary -dismember -v --help]

Algorithm:

A normal seispp stream file is a sequence of objects in a linear file - the streams data model.   This program supports only a files made up of a sequence of ThreeComponentEnsemble objects.   It will make one output file for each ensemble in the input file.  The output files will have names = basename_n where "basename" is the string entered as (required) argument 1 and n is the sequence number of each ensemble in the file (0, 1, 2, ..., n). 

The inverse of this program is cat_seispp.

Options:

-i
Sets the input to infile instead of default of stdin.

-dir
Writes the output files to the directory outdir.  Default is the current directory ("." in unix jargon).

-binary
Switch to binary input and output.  

-dismember
This is a switch that will ungroup each ensemble.   That is, the ensemble grouping is stripped and ensemble metadata are copied to each output seismogram.  The output files in this case a serialized ThreeComponentSeismogram objects instead of one ThreeComponentEnsemble per file. 

-v
Be more verbose in output (to stderr).

--help
Echo the usage line to stderr (equivalent to typing fragment with no arguments).
