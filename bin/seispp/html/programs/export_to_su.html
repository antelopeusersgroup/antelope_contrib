export_to_su

Usage:

export_to_su outfile [-pf pffile] < seispp_input

Algorithm:

This is a (fairly limited) program to export serialized seispp data to a format readable by the seismic unix package.   The input must be a TimeSeriesEnsemble object with constant start times for all seismograms in the ensembles and a constant number of samples for each seismogram.  This is a hard constraint by the SU format, which is a variant of SEGY, and is thus assumed to be valid.   Irregular start times will be silently ignored.   Irregular number of samples will cause the program to abort.  

Data are written to the file defined by the (required) outfile argument in SU format.  What attributes are transferred from the generic headers of the TimeSeries objects to the headers of the SU traces is controlled by the parameter file.

Options:

Use -pf to read from parameter file pffile instead of the default export_to_su.pf.

Parameter file


The parameter file has two Tbl lists that control what is written to SU headers.  
  1. The output_metadata_list tag defines a list of names to extract from the TimeSeries object Metadata (generic header).   The names must match Metadata tags found in each input seismogram or the program will fail.
  2.  How the Metadata attributes are translated to the SU namespace is defined by the metadata_cross_reference Tbl list.   Each line in this Tbl has this format:
internal_name SU_name type
where "internal_name" is the Metadata tag while "SU_name" is the name defined in the standard SU namespace (accessed for example by the sukeyword command).  "type" should be one of real, integer, or boolean.  Note string is not allowed since SEGY headers are purely numeric.
