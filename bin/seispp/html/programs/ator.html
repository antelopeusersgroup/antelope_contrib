ator

Usage:

ator < in > out [-key key -t objt --help -text -v]

Algorithm:

A novel feature of the seispp library is a consistent handling of absolute and relative time.  By absolute time I implicitly mean UTC specified as epoch time.  The concept, however, is more generic.  Any time standard that allows data obtained from multiple sources to be trusted as following the same standard are "absolute".  Said another way, absolute time is a clock standard.  Relative time, in contrast, is not a clock time but a time relative to some (unspecified) 0 time.   The definitive example is active source data where 0 is shot time.   In the seispp library, however, another useful standard I use a lot is an "arrival time reference" where 0 is the predicted or measured time of a "seismic phase" with respect to an event hypocenter.  The purpose of this program is to convert seismograms from absolute to a relative time base. 

The time used to define zero is controlled by a time extracted from the Metadata (header) of each and every seismogram.  The default key is "arrival.time".  Ensembles (gathers) must be passed through dismember before calling this program or processing will fail because the algorithm uses assumes an input stream of single seismogram objects. 

Options:

-key key
Use key as parameter defining the epoch time of time 0.  Default key is arrival.time.   Example of alternative names that might come from an antelope db are origin.time or wfdisc.time.  Any name can be used, however, so something simpler like t0 could be used provided all seismograms contain that attribute with that key.

-t objt
Use alternative object type objt instead of default (ThreeComponentSeismogram).  Accepted names are TimeSeries and PMTimeSeries.

-text
Write the output in text format.   Default is a binary serialized file.

-v
Be more verbose.  Note all log information goes to stderr so it is best to use this option only for debugging problems.

--help
Echo the usage line.
